# PR: vLLM Qwen3.5-27B + resumable auto-repair, grouped commands

## Summary
- **Default vLLM model:** `openai/gpt-oss-120b` → `Qwen/Qwen3.5-27B`. Tests updated; telegram two_tasks mock fixed.
- **Resumable tasks:** On failure in bot context, a long (resumable) repair task is created with full error context. SessionDB: `get_resumable_tasks(chat_id?, limit)`, `get_resumable_task_by_id(id)`.
- **Bot:** Commands and HELP grouped (main, tasks, session, logs, emergency). New `/longlist` — list long tasks for this chat.
- **CLI:** `agent long list|resume|show`, `agent session context|clear`; backward compat `context` / `clear-context` / `resume`.
- **Docs:** AGENTS.md — vLLM default Qwen, test and env vars.
